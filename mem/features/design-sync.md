---
name: Design sync rules
description: Figma 화면 맞춤 작업 시 반복 적용할 공통 규칙
type: feature
---
- Figma에서 공통 헤더로 보이는 화면은 기존 구현을 우선하지 말고 공통 헤더 구조를 맞춘다.
- 공유일기 계열 흰색 헤더 기준: 상단 흰 배경, 좌측 44px 뒤로가기 영역, 중앙 18px 타이틀, 하단 탭은 별도 흰색 바 + 선택 탭 파란 underline.
- 공유일기 리스트 공통 기준: 화면 배경 위 전체 폭 행, 좌우 24px, 56px 썸네일(12px radius), 15px bold 제목, 12px 회색 메타, 댓글 줄은 있을 때만 표시, 우측은 chevron이 아니라 kebab 메뉴, 행 하단 #ececec 구분선.
- 공유일기 등록 폼 기준: 공통 흰색 헤더, 본문 좌우 24px, 섹션 라벨 15px bold #222, input 53px 높이/10px radius/#e6e6e6 border, 이미지 업로드는 98px 정사각형 dashed tile, 하단 완료 버튼은 흰 배경 위 327px 폭/52px 높이/12px radius.
- 화면별 Figma 링크가 주어지면 텍스트, 순서, 섹션 제목 유무까지 같이 확인한다. "기능은 같은데 레이아웃만 다름"으로 넘기지 않는다.
- 공통 컴포넌트/패턴에서 벗어난 기존 구현을 발견하면 조용히 유지하지 말고, 무엇이 공통에서 벗어났는지 먼저 짚고 맞춘다.
